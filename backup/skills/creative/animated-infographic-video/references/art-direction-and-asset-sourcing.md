# Art direction and asset sourcing for reference-matched infographic explainers

Use this when the user wants a YouTube-style explainer, moving maps, newsroom-style war/geopolitics graphics, or explicitly rejects rough custom illustrations.

## Durable lessons
- Do **not** rely on rough hand-built placeholder illustrations for the final pass when the user is asking for premium, reference-matched motion graphics.
- Start asset sourcing early. For this class of work, check real vector/stock libraries before committing to a custom illustration pipeline.
- If Magnific is already logged in, verify access there first because it may expose Freepik-family stock inside the authenticated app even when generic browser access to Freepik is blocked by 403/security filters.
- For war/map explainers, prefer a **map-heavy newsroom style** over character-heavy editorial art unless the user explicitly asks for illustrated characters.

## Recommended source order
1. Magnific stock / illustrations / vectors (if logged in)
2. Storyset (customizable vector illustrations)
3. unDraw (clean free vectors)
4. OpenStreetMap base maps + icon packs / Noun Project-style pictograms
5. Premium libraries like Envato / Adobe Stock when available

## Verification checklist
- Confirm the user is actually logged in to the asset platform.
- Verify the stock hub/library page is reachable.
- Verify the relevant asset class exists: illustrations, vectors, icons, maps, templates, videos.
- Whenever possible, confirm a specific asset detail or download/export path before promising the library is fully usable.

## Visual direction for this task family
- Use real vector assets, clean map layers, route lines, callout boxes, and icon systems.
- Prefer simplified geopolitical maps, highlighted strike locations, weapon/system badges, shipping lanes, and clean labels.
- Avoid over-cluttered compositions and avoid obviously homemade placeholder art in the final render.

## Session note
In this session, the user explicitly said the prior graphics/illustrations were "very bad" and asked for real high-quality illustration sources. This preference should govern future infographic-video rebuilds of the same class.
