# Rendering Checklist for Short Animated Infographic Explainers

Use this as a compact preflight when building illustrated explainer videos with local tools.

## 1. Brief fit
- Is the user asking for motion rather than a static infographic?
- Did they reference a video style, map explainer, or animated information design?
- Is the runtime target explicit? If not, assume 35–60 seconds.

## 2. Fact grounding
- Verify place names and strike/route/chokepoint labels before rendering.
- Separate confirmed events from risk-language or scenario-language.
- For waterways/blockades/conflict claims, prefer carefully qualified wording unless closure/control is firmly established.

## 3. Scene plan
Aim for 4–7 scenes total.
For each scene, define:
- purpose
- key labels
- motion element
- panel/callout content
- narration line
- transition

## 4. Map scene heuristics
- Show the region first, then zoom attention to the active corridor/site.
- Use arrows and route lines before explanatory text paragraphs.
- Put narrow-waterway explanations in boxed callouts.
- Use icon shorthand (MISSILE, DRONE, SHIP, OIL, AIR, INS) instead of long bullets where possible.

## 5. Motion and readability
- Add camera drift, route growth, icon movement, or panel reveals so the video feels alive.
- Keep text blocks short and high-contrast.
- Design for phones: central safe area, generous font size, low edge dependency.

## 6. Audio timing
- Draft voiceover early.
- Measure duration.
- Shorten script before over-compressing visuals.
- Pad or trim final assembly so scene changes feel intentional.

## 7. Export
Preferred defaults:
- 1280x720 or 1920x1080 depending on render budget
- 24 or 30 fps
- H.264
- yuv420p
- AAC
- +faststart

## 8. Final QA
Before delivery, verify:
- representative frames are readable
- labels do not clip late in the scene
- map callouts still make sense after animation progress
- output codec is phone-safe
- runtime matches the brief closely
